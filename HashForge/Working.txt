-------------------------------
How the HashForge Tool Works:
-------------------------------

---------------------
User Interface Input:
---------------------
     -> The user enters a password hash they want to crack or a plaintext password to generate hashes.
     -> The user can optionally enter a salt value if they know the hash uses salted passwords.
     -> The user can input mutation rules (like capitalize, upper, reverse, leet) to try common password variations.
     -> The user selects a wordlist file containing candidate passwords.

--------------------
Hash Type Detection:
--------------------
     -> The tool attempts to automatically detect the hash type based on the hash’s format and length.
     -> For ambiguous hash types (such as 32-character hashes which could be MD5 or NTLM), it provides a dropdown to let users select the correct hash type manually.

-----------------
Cracking Process:
-----------------
     -> For dictionary attacks, the tool reads each word from the selected wordlist.
     -> It applies any specified mutation rules to each word, creating variants.
     -> For each word or variant, the tool hashes the candidate using the detected or selected hash function.
     -> If a salt is specified, it combines the salt with the candidate password before hashing.
     -> The computed hash is compared to the user-provided hash.
     -> For bcrypt, which uses an internal salt and specific verification, bcrypt.checkpw is used to verify candidate passwords correctly.
     -> If a match is found, the tool outputs the cracked password and stops.
     -> If no matches are found after exhausting the wordlist, it notifies the user.

----------------
Hash Generation:
----------------
     -> The user can enter any plaintext password.
     -> The tool computes and displays the hashes of that password across multiple algorithms (MD5, SHA variants, NTLM, base64 encoding, bcrypt) clearly formatted with spacing for easy copying.

----------------
User Experience:
----------------
     -> The GUI is split into two panels—one for cracking, one for generation—both visible and resizable.
     -> The interface provides real-time feedback, displaying detected hash types, chosen wordlist, and progress messages.
     -> All operations (especially cracking) are run in background threads to keep the interface responsive.
